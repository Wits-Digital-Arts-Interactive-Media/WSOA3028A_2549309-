Critical Reflection on My Design Process and Final Website

29 May 2024 by Aaminah Habib

Creating my first website was an exhilarating journey that transformed my initial vision into a polished digital space. This reflection explores the evolution of my project, the strategic decisions made to balance creativity with functionality, and the invaluable lessons learned.

I'm satisfied with how my final website turned out, though it evolved significantly from my initial vision. During my journey of learning CSS and exploring award-winning websites, I discovered the impressive possibilities of website design. Some of these features, which I initially wanted to include, had to be removed because I couldn't get them to work properly and be responsive as I wanted. For instance, I wanted to incorporate advanced parallax scrolling, where different elements move at varying speeds. This effect requires more JavaScript, which in turn demands more processing power and can increase load times, putting additional pressure on performance. As this is my first website, I decided to prioritize learning to do things right over being overly creative. By focusing on making the design responsive, ensuring faster load times, maintaining good performance speed, and adhering to good coding practices (such as using meta tags, microformatting, and semantic HTML), I aimed to build a solid foundation. This approach aligns with the principles I discussed in my blogs, particularly in the South African context, where mobile-first design is crucial due to the high mobile internet usage.

Coding practices

Reflecting on my coding practices for HTML and JavaScript, I feel confident that they were well-structured and efficient. However, my approach to CSS revealed significant areas for improvement, particularly in terms of scalability and reusability.

Instead of creating reusable classes for common elements such as headings, containers, and text, I ended up writing separate styles for each page (e.g., essay, blog, about). This approach led to unnecessary repetition and clutter in my CSS files.

Because I didn’t use reusable classes, making changes to the design required adjusting the styles for each individual element across multiple pages. This process was time-consuming and prone to errors, making it difficult to maintain and scale the website effectively. Through this experience, I’ve learned the importance of scalable and reusable CSS practices. Adopting these strategies not only streamlines the development process but also enhances the maintainability and flexibility of the website. Moving forward, I will prioritize creating reusable classes, utilizing CSS variables, adopting a modular approach, and leveraging preprocessors to write cleaner, more efficient CSS.

Improving my CSS practices is essential for building a robust, maintainable, and scalable website. By focusing on reusability and scalability, I can ensure that my website remains flexible and easy to update, providing a better experience for users and a more efficient workflow for myself. This reflection has been an invaluable part of my learning journey, and I look forward to implementing these best practices in future projects.

During the planning phase of my project, I was introduced to the DRY (Don't Repeat Yourself) principle, a core concept in software development that emphasizes reducing repetition in code. The idea is to ensure that every piece of knowledge or logic is represented in a single, unambiguous place, which can significantly enhance the maintainability and readability of a codebase. As I worked on implementing my project, I incorporated this principle into my style guide, aiming to write clean and efficient code. However, as I progressed, I realized that I had considerable repetition in certain areas, particularly in elements like breadcrumbs and menus. Despite my initial understanding of the DRY principle, I found myself duplicating code across various parts of the project. Reflecting on this, I recognize that my ability to fully adhere to the DRY principle was limited by my current skill level and the time constraints I faced. While I understood the importance of minimizing repetition, I struggled to implement it effectively. For example, instead of creating reusable components or functions for breadcrumbs and menus, I ended up copying and pasting similar code in multiple places. Fixing these issues and fully applying the DRY principle would have made the code more effective. By reducing the amount of JavaScript, it could have made the site load faster and improve overall performance. Less repetitive code means fewer lines of JavaScript for the browser to interpret and execute, leading to quicker load times and a more responsive user experience. Upon this realization, I acknowledged that revising the entire codebase to eliminate these repetitions was not feasible given the time left in the project. This experience has highlighted the importance of not only understanding key programming principles but also having the practical skills to implement them efficiently. It has also underscored the need for better planning and foresight in future projects to identify areas where the DRY principle can be applied from the outset.

GitHub

My step for designing the website was setting up GitHub. The software I feared! At the beginning, navigating GitHub was challenging. The command-line interface was unfamiliar, and I worried about making errors that could break my site or lose my work. The concept of version control, while theoretically understood, seemed complex in practice. It was especially scary to push changes and view live pages, fearing that something might go wrong.

Gradually, as I started using GitHub more frequently, the fear began to subside. Embracing GitHub has been a significant milestone in my development as a web developer. It has taught me the importance of version control, disciplined me to document my progress meticulously, and equipped me with skills that are highly valued in the tech industry.

In conclusion, what once seemed like an enemy has become a trusted ally. GitHub is now a tool that I rely on for managing my projects efficiently and professionally. The initial fear has transformed into confidence, underscoring the fact that overcoming challenges is a crucial part of growth in any learning journey.

Overall Aesthetics

As I reflect on my portfolio website, I see that it looks very structured, minimalistic, and basic. I avoided adding any fancy, extra elements. While it might appear characterless, professional, and strict, it reflects a certain aspect of myself. It demonstrates that I am simple, professional, clean, focused, and directed, without distractions or unnecessary elements compromising usability.

This minimalistic approach aligns with my overall design philosophy. By keeping the design clean and straightforward, I ensure that the user experience remains focused on the content without any distractions. Although the website might lack some creative flair, it serves its purpose well by being user-friendly and efficient. This decision reflects my commitment to creating practical, functional designs that prioritize usability over unnecessary embellishments.

In developing my website, I extensively researched best practices in UI and UX design, drawing from a variety of academic sources and industry experts. For example, Don Norman's principles of design, particularly the importance of usability and user-centered design, significantly influenced my approach. Norman emphasizes that good design is centered around the needs and abilities of users, which aligns with my decision to prioritize a responsive, mobile-first design for my South African audience (Norman, 2013). Research from Jakob Nielsen also reinforced the importance of simplicity and ease of use, guiding my minimalist approach to ensure users can easily navigate and engage with the content (Nielsen, 1994).

Furthermore, the concept of "Progressive Enhancement" played a crucial role in my design strategy. Progressive enhancement focuses on building a basic, functional version of the site that works for all users, and then adding advanced features for users with more capable devices and browsers. This ensures that my site remains accessible on low-bandwidth connections while still providing a rich experience for those with better internet access (Keith, 2010).

Given that many South Africans access the internet primarily through their smartphones, I adopted a mobile-first approach. This decision is supported by data from the International Telecommunication Union (ITU), which highlights the high mobile internet usage in the region (ITU, 2021). Designing with a mobile-first mindset ensured that my site performs well on smaller screens and touch devices, which is crucial for reaching a wider audience. However, I also ensured that the desktop experience remains rich and engaging, striking a balance between creativity and functionality.

I realized late in the development process that I had overlooked important design considerations, particularly regarding spacing. I ignored the spacing of texts, lines, and paragraphs, which resulted in inconsistent and sometimes cramped layouts. There was also a significant amount of empty white space under the navigation before the main headings, especially when viewed on a phone. This negatively impacted the overall user experience, making the site feel less cohesive and more difficult to navigate. Addressing these spacing issues would have improved the visual appeal and usability of the site. Proper spacing ensures better readability and a more professional look, while reducing excessive white space can make the site feel more balanced and engaging.

Other than that, I managed to stick to my style guide. Maybe not the colour though. However most of it was implemented and I can proudly say that I think I have achieved my goal I outlines and managed to create the intended user experience.
